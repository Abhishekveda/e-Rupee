Agentic AI

A ReAct agent that reasons,
plans, and acts — no black box.

Unlike a simple API wrapper, this is a genuine agentic system. It classifies intent, selects tools, calls them, observes the results, and synthesises a complete answer. Every step is logged.

INPUT
Security Filter → Input Sanitiser → Rate Limiter → Injection Detector
Every input is cleaned and checked before the agent sees it
↓
PLAN
Intent Classifier → Tool Planner (ReAct loop: Thought → Action → Observe)
Agent decides which tools to call and in what order
↓
TOOLS
FEMA Agent · Risk Scoring Agent · Regulatory Q&A Agent · LRS Calculator
Deterministic tools — auditable, explainable, callable by the planner
↓
ANSWER
Response Synthesis → Memory Update → Audit Log Entry
LLM formats the answer if available; rule synthesis used as fallback
01 / 03 · FEMA AGENT
⚖️
FEMA Classification Agent
Classifies plain-English descriptions into FEMA purpose codes using TF-IDF cosine similarity across 13 codes. Catches paraphrases, not just keywords.
input: "paying daughter's MBA fees in Dubai" output: P0103 · HIGH confidence · 0.82 score method: keyword match + TF-IDF similarity
✓ No external API
02 / 03 · RISK AGENT
🛡️
Risk Scoring Agent
Scores every transfer 0–100 before execution. Eight named, transparent rules. Every flag has a named reason an auditor can read.
rules: 8 named (LRS_LIMIT, HIGH_VELOCITY...) output: score 5/100 · LOW · APPROVE maps to: RBI KYC Master Direction
✓ FIU-IND aligned

AI Security

Every input is checked.
Every decision is logged.

The security layer sits between the user and the agent. It blocks adversarial inputs, rate-limits abusive sessions, and maintains a tamper-evident audit chain that the RBI can inspect.

🔒
Prompt Injection Guard
30+ injection patterns detected (ignore instructions, role override, system prompt extraction)
Session blocked immediately after detected injection attempt
Input sanitised: control characters removed, truncated to 2,000 chars
Based on published LLM adversarial attack research
📋
Immutable Audit Chain
Every agent interaction logged: input hash, output hash, tools used, timestamp
Entries are hash-chained — tampering breaks the chain and is detectable
Chain integrity verifiable via GET /v1/security/audit-log
In production: append-only PostgreSQL with row-level checksums
⚡
Rate Limiting
Per-session token bucket: 30 requests per 60 seconds
Configurable limits per deployment environment
Rate limit events logged to audit trail
Remaining request count returned in every response
🌐
Data Sovereignty
Logic-only and Ollama modes process all data locally — no external network calls
Sarvam AI (Indian LLM) processes data on Indian infrastructure
Groq mode clearly documented as US-based; not for live customer data in RBI context
Architecture fully compliant with India's data localisation requirements
